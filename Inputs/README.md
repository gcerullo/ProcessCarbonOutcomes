# Inputs Required For End-To-End Carbon Pipeline

This document lists the inputs needed to run the full Nature Revision 2 carbon
workflow in `Scripts/Nature_Revision_2/` from start to finish.

It is written so the project is self-contained operationally: if a collaborator
has these files in the expected locations, they can run all steps.

---

## 1) External Input Files You Must Provide

### RawData (used by step 01 model fitting)

Place these in `RawData/`:

- `SSB_plot_data.csv`
- `Philipson20_PlotData2.csv`

Used by:
- `Scripts/Nature_Revision_2/01_one_model_to_rule_them_all.R`

### Inputs (used by steps 03 and 04)

Place these in `Inputs/`:

- `Venmans_Value of an offset 25_1evening.csv`
  - Used by step 03 (`03_calculate_social_discount_rates.R`)
  - Provides temperature trajectory inputs for SCC discount curves.

- `FixedScenarioParmams.R`
  - Used by step 04 (`04_propagate_carbon_thru_scenarios.R`)
  - Must define object `all_start_landscape` when sourced.

- `MasterAllScenarios.rds`
  - Used by step 04 for scenario composition summaries/plot metadata.

- `MasterAllScenarios_withHarvestDelays.rds`
  - Used by step 04 as the main scenario schedule list.

---

## 2) Inputs Generated By Earlier Pipeline Steps

These are not manual data files, but they are required by downstream steps.

- Step 01 writes:
  - `Outputs/Nature_Revision_Outputs/NR2/current/01_one_model/models/unified_linear_carbon_model.rds`

- Step 02 reads that model and writes:
  - `Outputs/Nature_Revision_Outputs/NR2/current/02_draws/rds/acdraws_aboveground.rds`

- Step 03 writes:
  - `Outputs/Nature_Revision_Outputs/NR2/current/03_scc/tables/scc_dr_2_4_6.csv`

- Step 04 (`04_propagate_carbon_thru_scenarios.R`) requires both:
  - `02_draws/rds/acdraws_aboveground.rds`
  - `03_scc/tables/scc_dr_2_4_6.csv`

---

## 3) Minimal Structure Expectations (Step 04)

### `FixedScenarioParmams.R`

When sourced, it must create:
- `all_start_landscape`

And `all_start_landscape` should contain at least:
- `habitat`
- parcel-count/area fields needed by your scenario baseline logic

### `MasterAllScenarios.rds`

Expected as a list of scenario tables with columns used for composition:
- `scenarioName`
- `index`
- `production_target`
- `habitat`
- `num_parcels`

### `MasterAllScenarios_withHarvestDelays.rds`

Expected as a list of scenario schedules with columns used in transition rules:
- `scenarioName`
- `index`
- `production_target`
- `habitat`
- `original_habitat`
- `harvest_delay`
- `true_year`
- `num_parcels`

Additional columns are fine and are preserved when possible.

## 4) Quick Pre-Run Input Check (optional)

From repo root in R:

```r
required_files <- c(
  "RawData/SSB_plot_data.csv",
  "RawData/Philipson20_PlotData2.csv",
  "Inputs/Venmans_Value of an offset 25_1evening.csv",
  "Inputs/FixedScenarioParmams.R",
  "Inputs/MasterAllScenarios.rds",
  "Inputs/MasterAllScenarios_withHarvestDelays.rds"
)
missing <- required_files[!file.exists(required_files)]
if (length(missing)) stop("Missing input files:\n", paste(missing, collapse = "\n"))

env <- new.env(parent = emptyenv())
sys.source("Inputs/FixedScenarioParmams.R", envir = env)
if (!exists("all_start_landscape", envir = env, inherits = FALSE)) {
  stop("FixedScenarioParmams.R does not define all_start_landscape")
}
```

## 5) Run Order

Recommended full run order:

1. `01_one_model_to_rule_them_all.R`
2. `02_prepare_draws.R`
3. `03_calculate_social_discount_rates.R`
4. `04_propagate_carbon_thru_scenarios.R`

Or run the pipeline driver:

```r
source("Scripts/Nature_Revision_2/run_all.R")
```

If any path/filename changes, update the corresponding `file.path(...)` calls in
the relevant step script.
